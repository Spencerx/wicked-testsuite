How to create a new test suite for Wicked

  1. Log into the server as an identified user.
  2. Go to Test suites tab.
  3. On the left, select New Item.
  4. A form appears.
    • In the Name field, type wicked-branch or wicked-branch-nanny.
    • Press the Copy existing item radio button.
    • In the Copy from field, select either wicked-master or wicked-master-nanny.
    • Press OK.
  5. The new test suite is created, and you are presented with another dialog page to set up its characteristics.
    • In the Description field, it is interesting to mention who you are, he name of your branch, and whether nanny is enabled or not.
    • In the Source code management / Repositories, type in your git URL, like https://github.com/yourname/wicked.git.
    • In the Source code management / Branches to build, type in your branch specification, like */branchname.
    • In the second Build / Execute shell script input area, check on the last line whether it's wicked-with-nanny.xml or wicked-without-nanny.xml that is used.
    • Validate and exit with Save button.

A new build should start automatically within 10 minutes. To make it happen immediately, you can use the Start build link on the test suite's page.

It is nice also to remove old test suites that you do not need anymore.
